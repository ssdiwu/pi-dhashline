# 初版架构

## 目标

以零运行时依赖的 Pi 原生扩展提供 `read`、`edit`、`write`、`search`，使读取与搜索结果直接成为安全编辑锚点，并阻止 `write` 绕过已有文件的标签保护。

## 调用链

```text
read/search
  → 读取完整文本（最大 4 MiB）
  → 规范化 BOM 与行尾
  → 计算 8-hex 文件标签
  → 记录 session 快照与已见行
  → 用压缩 custom entry（自定义会话条目）持久化快照/观察
  → 输出 [PATH#TAG] + LINE:TEXT

edit
  → 解析单文件文本协议
  → 进入 withFileMutationQueue
  → 读取并规范化当前文本
  → 校验 live tag；不一致时查 session 快照并保守恢复
  → 校验范围、重叠与已见行
  → 内存应用全部操作
  → 原子写入并保留文件语义
  → 记录新快照，返回新标签与 diff
```

write
  → 解析原生 path + content 参数
  → 进入 withFileMutationQueue
  → 校验容量、Unicode 文本与二进制控制内容
  → 要求父目录已存在且不是符号链接，在其 canonical path 内准备完整临时文件
  → 以 hard-link create 原子发布目标；目标已存在或为符号链接时拒绝，不覆盖任何字节
  → 读回实际文本，记录 fresh snapshot 但不授予已见行
  → 返回新标签、行数与字节数

## 模块职责

- `config.ts`：读取并严格校验项目根 `dhashline.json`。
- `hash.ts`：文件规范化标签。
- `snapshots.ts`：按 session 与 cwd 隔离的多版本 LRU 快照与已见行。
- `protocol.ts`：文本协议解析、冲突检查、应用和保守恢复。
- `files.ts`：路径、文本格式、仅创建写入、原子替换与精简 diff。
- `tools.ts`：Pi `read` / `edit` / `write` / `search` 定义和宿主适配。
- `tui/tool-result.ts`：四工具的默认摘要、动态快捷键展开和可操作错误投影。
- `index.ts`：注册工具与 `/dhashline` 诊断命令。

## 协议约束

- 文件 header 必须是 `[PATH#8_HEX_TAG]`。
- 初版每次 `edit` 只允许一个文件段。
- `SWAP` / `DEL` 范围为原始文件中的闭区间。
- `SWAP` / `INS` 的 body 每行必须以 `+` 开始；`+` 自身表示空行。
- 所有操作先校验且不得重叠，再按原始坐标从后向前应用。
- 未见行、越界、标签未知、目标内容变化、标签碰撞或恢复歧义均拒绝写入。
- 任一 `SWAP` 若已经等于当前内容，整批请求在落盘前拒绝；模型必须以最新 read/search 为准并省略已满足操作，不能继续执行同批插入。
- 纯 HEAD / TAIL 操作在 stale 状态没有内容锚点，必须拒绝。
- edit 结果的 renderer details（渲染详情）不视为模型已见；fresh tag 必须重读后再编辑。
- write 结果的 fresh tag 同样不视为模型已见；必须重读后才能编辑。
- write 只允许创建不存在的文本文件；目标存在、符号链接、容量超限、非法 Unicode 或二进制控制内容均在不覆盖目标的前提下拒绝。

## 依赖边界

`package.json` 不声明 runtime `dependencies`。运行时只使用：

- Node 标准库；
- Pi 提供的 `@earendil-works/pi-coding-agent`；
- Pi 提供的 `@earendil-works/pi-tui` 原生组件；
- Pi 提供的 `typebox` schema（模式）包。

`search` 复用 Pi `createGrepToolDefinition`；其文本输出解析是受定向测试保护的宿主兼容接缝。有效 UTF-8 是可编辑前提；符号链接写入真实目标，硬链接因“链接身份”与“rename 原子替换”不可同时保证而拒绝。

## Session 生命周期

快照内容使用 Node gzip（压缩）写入不参与 LLM 上下文的 Pi custom entry；已见行另以小条目追加。`session_start` 按当前 branch（分支）条目重建 LRU，因此 resume / fork / reload 后仍保持标签与已见行隔离。`/dhashline clear` 写入清除标记，避免旧条目在后续 reload 中复活。

## 人类可读投影

模型始终接收精确协议 `content`；人类 UI 通过独立 renderer 投影。默认态只呈现动作、对象、数量、状态和 tag。展开态中，read 只显示窗口首尾各一行，edit 每个 diff hunk 只保留修改前后各一行，write 只显示创建元数据，search 保留稀疏匹配锚点；错误展开给出原因、未写入状态和下一步，不能把原始 JSON 或 details 交给默认 renderer。

## 测试层次

1. 单元：标签、解析、操作应用、冲突、恢复和两层工具投影。
2. 集成：真实临时文件、仅创建/原子替换、快照和四工具调用链。
3. 冒烟：Pi 0.80.10 以 RPC（远程过程调用）模式实际加载扩展。
