# ADR 0002：采用文件级八位标签与单文件文本协议

## 状态

Accepted

## 背景

逐行短哈希允许局部稳定，但协议需要为每行支付 token（令牌）成本并承担重复行定位问题。oh-my-pi 的成熟实现转向全文件标签和裸行号，但 4 位十六进制标签只有 16 bit（位）空间。

## 决策

- 使用规范化完整文件的 SHA-256 前 8 位大写十六进制作为文件标签。
- `read` / `search` 输出 `[PATH#TAG]` 与 `LINE:TEXT`。
- `edit` 使用单文件文本协议，初版包含 `SWAP`、`DEL`、`INS.PRE`、`INS.POST`、`INS.HEAD`、`INS.TAIL`。
- 标签不匹配时只允许基于 session 快照的保守恢复；不做模糊近邻重定位。

## 权衡

- 8 位标签比 4 位多 4 个字符，但将随机碰撞空间从 16 bit 提升到 32 bit。
- 文件任意变化都会使标签过期；多版本快照和保守恢复负责吸收不影响目标内容的位移。
- 文本协议比结构化 JSON 更紧凑，但需要自维护解析器与强行为测试。

## 结果

改变标签长度或操作语法会使已有会话锚点失效，必须作为 breaking change（破坏性变更）发布。
