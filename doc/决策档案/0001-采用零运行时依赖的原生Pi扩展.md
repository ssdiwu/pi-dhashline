# ADR 0001：采用零运行时依赖的原生 Pi 扩展

## 状态

Accepted

## 背景

现有 Hashline 实现分别依赖 Bun 或第三方 hash、diff、MIME 与缓存包。直接依赖能减少初始代码，但扩大供应链、运行时和版本兼容面。

## 决策

`pi-dhashline` 的 runtime `dependencies` 保持为空。实现只使用 Node 标准库与 Pi 宿主提供的公开 peer dependencies（coding-agent、pi-tui、typebox）。模型工具固定命名为 `read`、`edit`、`search`，不以实现前缀污染能力名称。

## 权衡

- 优点：安装简单、审计面小、与 Node 版 Pi 一致、无 Bun 接缝。
- 代价：需要自行维护标签、快照、协议、diff 和原子写入逻辑；Pi 内部输出变化需要兼容测试。

## 结果

任何新增运行依赖都必须以新 ADR 说明无法由标准库或 Pi 能力完整满足的真实问题。
