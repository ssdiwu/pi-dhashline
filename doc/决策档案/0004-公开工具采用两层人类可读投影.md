# ADR 0004：公开工具采用两层人类可读投影

## 状态

Accepted

## 背景

真实 Pi 测试中，`read` / `search` 的模型结果可驱动编辑，但三个工具的 UI 展示不一致；`edit` 忽略 `Ctrl+O` 展开状态，错误只返回原始 parser 文案。更严重的是 edit 提示没有给出 `INS.POST N:` 等完整语法，导致模型连续四次猜错并把一次原子编辑拆成两次。

## 决策

- `read`、`edit`、`write`、`search` 的公开工具结果统一使用两层人类可读投影和 Pi 标准工具外壳 `renderShell: "default"`。
- 调用与结果共享同一个有状态 TUI 组件：执行前显示动作，完成后原位更新为结果，不叠加第二行重复工具标题。
- 组件使用 Pi `theme`、`Text` / `Container`、`keyHint` 和 `renderDiff`，遵守宿主快捷键、换行、宽度、成功/错误背景与主题切换约定；图片继续委托内置 `read` renderer。
- 默认视图只显示工具动作、对象、关键数量、状态和 tag，并提示 Pi 当前展开快捷键。
- 展开视图只保留必要审计证据：`read` 显示窗口首尾各一行，`edit` 的每个 hunk 显示修改前后各一行，`write` 显示创建元数据而不回显正文；共同保留告警和下一步。
- 错误默认收敛为一行摘要；展开时显示收到的错误输入、原因和可复制的正确语法。
- 模型可见 `content` 继续保留精确 `[PATH#TAG]` 与行锚点；人类投影裁剪不改变模型上下文和 details 契约。
- edit 的公开描述必须逐项列出所有合法语法，不允许模型依赖错误试探。

## 权衡

- 优点：日常界面克制且一致，展开后可审计；模型首次调用成功率提高；错误可直接恢复。
- 代价：每个公开工具都要维护摘要与展开两层文案，并用测试防止 UI 与协议漂移。

## 结果

新增独立 TUI 投影模块和默认/展开/错误行为测试。工具协议、快照、安全校验和模型上下文保持不变。
