# 经验笔记

## 重跑人工编辑测试前必须重置 fixture

**现象**：测试提示要求把 `two` 改成 `TWO` 并在第 3 行后插入 `four`，但 fixture 已被上一轮修改为 `TWO` 且已有 `four`。高 thinking 模型在“用户旧描述”和 `read` 现实之间反复权衡 1704 reasoning tokens，最终仍重复插入一行 `four`。

**做法**：每轮人工测试先重置 fixture；模型提示与工具 guideline 都声明最新 `read` / `search` 为权威。协议层对已满足 `SWAP` 做整批预检拒绝，确保同批其他操作不能因旧测试描述继续落盘。

**证据**：session `2026-07-19T11-47-54-947Z_019f7a34-49c3-7b05-9143-d3c3f9e33636.jsonl` 中 read 返回 `2:TWO`、`4:four`，随后 edit diff 新增 `+5 four`。
